However, it will not process the services that have the loadBalancerClass field set in the service specification. The Load Balancer Class supports all the Custom Resources (such as VirtualServer, TransportServer, and IngressLink) and the loadBalancer service. This feature cannot be disabled. You need to either remove the loadBalancerClass field from the service or set the CIS deployment parameter load-balancer-class to the same value as the loadBalancerClass field in the service.
