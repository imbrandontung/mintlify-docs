### 為什麼 Agent 要長在現場？

**好的 AI Agent 不是寫出來的，是從問題現場長出來的。**

PPT 顧問畫的 Agent 漂亮但跑不動；遠端工程師寫的 Agent 解的是想像中的問題。在現場待過的人才知道：哪一通告警才是真的、哪一封 mail 該優先回、哪一條 SOP 是給稽核看的、哪一條才是真的能用。

我同步走 [2nd-ML100Days](https://github.com/imbrandontung/2nd-ML100Days)，補齊 ML 工程基本功：因為**懂 ML 才能讓 Agent 不只是 prompt 串接，而是真的會判斷、會學、會用**。

### Why agents grow on the ops floor

**Good AI agents aren't written. They grow from the problem floor.**

Agents drawn in slide decks look pretty but never run. Agents written by engineers who've never sat the ops chair solve imagined problems. Only people who've worked the floor know which alert is the real one, which email actually needs a reply today, which SOP exists for the auditor and which one operators actually run.

I work the [2nd-ML100Days](https://github.com/imbrandontung/2nd-ML100Days) challenge in parallel — because **understanding ML is what turns prompt-chaining into agents that actually judge, learn, and act**.
