# Security Policy

## 中文版

### 通報資安疑慮 / Report a Security Concern

如發現本站或本 repo 內容有以下情況，請透過 **LinkedIn 私訊**（[imbrandontung](https://www.linkedin.com/in/imbrandontung/)）通報：

- 不慎洩露的個資、機敏資料、或客戶識別資訊
- 任何 mdx / json / 設定檔內的可疑連結或內容
- 偽冒本站的釣魚網站
- 偽冒站主身份的社交帳號

### 我的承諾

- **24 小時內回覆**確認收到
- **72 小時內**評估與處置
- 重大問題立即下架相關內容
- 通報者匿名保密（除非你同意公開致謝）

### 不在範圍內

- Mintlify 平台本身的漏洞 → 請直接通報 [Mintlify Security](https://mintlify.com/security)
- GitHub 平台漏洞 → [GitHub Bug Bounty](https://bounty.github.com/)
- 第三方函式庫漏洞 → 對應上游 repo

## English

### Report a Security Concern

If you spot any of the following on this site or repo, please report via **LinkedIn DM** ([imbrandontung](https://www.linkedin.com/in/imbrandontung/)):

- Inadvertent disclosure of personal data, sensitive information, or customer-identifying details
- Suspicious links or content inside any mdx / json / config file
- Phishing sites impersonating this site
- Social accounts impersonating the site owner

### My commitments

- **Acknowledgement within 24 hours**
- **Assessment and remediation within 72 hours**
- Critical issues will trigger immediate takedown of affected content
- Reporter identity kept confidential by default (named credit only with consent)

### Out of scope

- Mintlify platform vulnerabilities → report to [Mintlify Security](https://mintlify.com/security)
- GitHub platform vulnerabilities → [GitHub Bug Bounty](https://bounty.github.com/)
- Third-party library vulnerabilities → upstream repository
